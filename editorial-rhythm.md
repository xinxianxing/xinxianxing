# 信先行编辑发布节奏

目标是让网站保持人工审核的可信度，同时不让草稿长期堆积。这个流程
只复用现有草稿、发布命令和 Git 提交，不新增后台或自动发布。

## 每日 10 分钟

每天早上打开审核群中的当天预览链接，或阅读：

```text
docs/_drafts/YYYY-MM-DD-summary-zh.md
```

按下面顺序快速检查每张卡片：

1. 标题和一句话简介是否准确、自然，没有翻译腔。
2. “具体怎么做”是否只来自原文；原文没有步骤的地方没有被补写。
3. 变现案例是否有数据依据，并保留风险或幸存者偏差提示。
4. 是否值得放进信先行公开档案，而不只是“今天可以推送”。

当天决定公开发布时，执行：

```bash
uv run xinxianxing-publish --date YYYY-MM-DD --language zh
git add docs/_posts/YYYY-MM-DD-summary-zh.md
git commit -m "content: publish 信先行 YYYY-MM-DD"
git push
```

`xinxianxing-publish` 会复制审核草稿，不会删除草稿、不会开启自动发布。

## 每周目标

- 每周至少发布 3 到 5 期中文精选。内容不足时少发，不为凑数量发布。
- 每周固定一次处理积压草稿：优先看评分高、被收藏或与你判断一致的内容。
- 当未发布草稿超过 7 天时，先决定“发布”或“保留但不公开”；不要把首页
  更新拖成一次性大清理。

## 周末复盘

查看当天或最近一周的：

```text
data/reports/xinxianxing-YYYY-MM-DD-zh-operations.json
```

重点记录三件事：

1. 哪些信源与栏目反复进入每日精选。
2. 哪些卡片被点了“有用”或“收藏”。
3. 哪些栏目连续没有内容，是否需要调整信源、分数门槛或暂时不推送。

这份复盘用于下一周的编辑判断；不要因为单日反馈就频繁改 Prompt 或评分。
